# Work Better - 架构总览

> **维护说明**：当系统架构、模块划分、层间关系发生变化时更新本文档。
> 本文档描述高层架构，各模块的详细设计见子文档。

## 架构原则

1. **分层解耦**——四层架构，每层只依赖下层，不跨层调用
2. **事件驱动**——采集层输出标准化事件，处理层消费事件，消除直接耦合
3. **单一真相源**——Obsidian 是所有数据的最终归属，其他存储层是派生索引
4. **可审计**——处理过程的每一步都产生审计记录，支持追溯和优化

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        呈现层                                │
│   菜单栏(即时信息) · 主窗口(深度交互) · Obsidian(最终视图)     │
│   速记窗口(全局快捷键) · 系统通知                             │
├─────────────────────────────────────────────────────────────┤
│                        存储层                                │
│   Obsidian Vault(文档) · 向量DB(RAG) · SQLite(结构化)        │
│   信息保鲜引擎 · 三层一致性保障                               │
├─────────────────────────────────────────────────────────────┤
│                        处理层                                │
│   分类器 · 四条处理路径(即时/聚合/模式/归档)                   │
│   小模型→大模型升级 · 审核代理 · 全链路审计                    │
│   SLA 时效管理 · 模型调度与 token 预算                        │
├─────────────────────────────────────────────────────────────┤
│                        采集层                                │
│   飞书采集器(可热插拔,多级开关) · 系统行为采集器               │
│   用户手动采集器 · EventLog(有序不可变)                       │
├─────────────────────────────────────────────────────────────┤
│                      定时任务系统                             │
│   跨层调度 · 错峰执行 · 依赖管理 · 执行日志                   │
└─────────────────────────────────────────────────────────────┘
```

## 数据流

```
信息源（飞书/系统/手动）
    │
    ▼
采集层 → EventLog（有序不可变）
    │
    ▼
处理层（分类 → 提取 → 审核）→ WorkRecord
    │
    ▼
存储层（Obsidian ← 向量DB ← SQLite）
    │
    ▼
呈现层（菜单栏 / 主窗口 / Obsidian）
```

## 层间通信

| 调用方向 | 机制 | 说明 |
|---------|------|------|
| 采集层 → 处理层 | EventLog（异步） | 事件写入日志，处理层订阅消费 |
| 处理层 → 存储层 | 直接调用（同步） | WorkRecord 写入三层存储 |
| 存储层 → 呈现层 | 查询接口（同步） | 呈现层从存储层读取数据展示 |
| 呈现层 → 存储层 | 写入接口（同步） | 用户编辑内容写回 Obsidian |
| 呈现层 → 采集层 | 命令接口（同步） | 用户手动触发采集（速记窗口） |
| 定时任务 → 各层 | 调度接口（异步） | 定时触发各层的周期性任务 |

## 私有 vs 共享边界

```
私有数据（AI 完全自主）：
  - 用户个人的任务、笔记、速记
  - Obsidian 中的所有文档
  - 审计数据、模型配置

共享数据（操作需用户确认）：
  - 飞书项目状态变更
  - 团队任务修改
  - 涉及他人的信息处理结果
```
