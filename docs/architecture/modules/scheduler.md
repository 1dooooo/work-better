# 定时任务系统架构

> **维护说明**：当新增/移除定时任务、修改调度策略或依赖关系时更新本文档。
> 定时任务的变更需同步更新 [功能索引](../../features/index.md)。

## 概述

定时任务系统横跨采集、处理、存储、呈现四层，负责调度所有周期性任务。

存储层的 S-01 到 S-09 任务由 [FreshnessEngine](storage.md) 实现，Scheduler 只负责调度触发，不关心具体执行逻辑。

## 调度器设计

```
Scheduler
  │
  ├── 任务注册
  │     ├── 每个定时任务实现统一接口
  │     ├── 支持运行时注册/注销（热插拔）
  │     └── 任务配置持久化到本地
  │
  ├── 调度策略
  │     ├── 基于 cron 表达式
  │     ├── 错峰执行
  │     ├── 依赖管理
  │     └── 用户活跃回避
  │
  ├── 执行管理
  │     ├── 超时控制
  │     ├── 重试策略（最多 3 次，间隔递增）
  │     ├── 并发控制（同类任务不并行）
  │     └── 资源感知（token 预算）
  │
  └── 执行日志
        └── 每次执行记录状态和结果
```

### 错峰策略

```
采集类（高频）→ 整点后 0-5 分钟
处理类（中频）→ 整点后 5-30 分钟
存储类（低频）→ 凌晨 02:00-05:00
报告类（按需）→ 用户设定时间
```

## 任务接口

```
interface ScheduledTask {
  id: string                // 任务 ID
  name: string              // 显示名称
  layer: 'collection' | 'processing' | 'storage' | 'presentation'
  cron: string              // cron 表达式
  sla_ms: number            // SLA 超时时间（毫秒）
  dependencies: string[]    // 依赖的任务 ID 列表
  retry_limit: number       // 最大重试次数（默认 3）

  // 执行
  execute(): Promise<TaskResult>
  abort(): void             // 终止正在执行的任务
}
```

### TaskResult

```
TaskResult {
  task_id: string
  status: 'success' | 'failed' | 'timeout' | 'aborted'
  started_at: datetime
  finished_at: datetime
  duration_ms: number
  summary: string           // 执行摘要
  error: string | null      // 失败原因
  retry_count: number       // 已重试次数
}
```

## 全局暂停与紧急停止

```
系统状态控制：
  ├── 正常运行：所有定时任务按计划执行
  ├── 全局暂停：暂停所有定时任务（用户手动触发）
  ├── 紧急停止：立即终止正在执行的任务 + 暂停所有队列
  └── 恢复：从暂停状态恢复，积压任务按优先级补执行
```
