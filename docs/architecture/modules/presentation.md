# 呈现层架构

> **维护说明**：当 UI 组件、交互流程或系统级快捷键变更时更新本文档。
> 呈现层的新增功能需同步更新 [功能索引](../../features/index.md)。

## 概述

呈现层是用户与系统交互的界面。它有三个入口，各司其职。

## 三个入口

```
┌─────────────────────────────────────────────────────────────┐
│                       呈现层                                 │
│                                                             │
│  ┌─────────────┐  ┌─────────────────┐  ┌─────────────────┐ │
│  │   菜单栏     │  │    主窗口        │  │   Obsidian      │ │
│  │             │  │                 │  │                 │ │
│  │ 即时信息     │  │ 深度交互         │  │ 数据最终视图     │ │
│  │ 快速操作     │  │ 多维展示         │  │ 长期存储         │ │
│  │ 被动通知     │  │ 主动探索         │  │ 自由组织         │ │
│  │             │  │                 │  │                 │ │
│  │ 交互频率：高  │  │ 交互频率：中     │  │ 交互频率：低     │ │
│  │ 停留时间：秒  │  │ 停留时间：分钟   │  │ 停留时间：分钟   │ │
│  └─────────────┘  └─────────────────┘  └─────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

## 菜单栏

常驻系统菜单栏，快捷键呼出，处理完即走。

### 功能模块

```
菜单栏
  │
  ├── 通知中心
  │     ├── 待确认事项（需要人工介入的操作）
  │     ├── 今日待办（截止日临近的任务）
  │     ├── 未读摘要（处理层生成的重要信息摘要）
  │     └── 保鲜异常（信息质量告警）
  │
  ├── 快捷操作
  │     ├── 快速记录（呼出速记窗口）
  │     ├── 手动触发处理
  │     └── 标记/收藏当前上下文
  │
  └── 状态概览
        ├── 今日已处理事件数
        ├── 待处理队列长度
        └── 系统状态（采集器健康、飞书连接）
```

### 设计原则

- 信息密度高，一屏看完所有需要关注的内容
- 操作不超过两次点击
- 不做复杂展示——复杂内容引导到主窗口

## 主窗口

独立窗口应用，用于深度交互和多维数据展示。

### 功能模块

```
主窗口
  │
  ├── 时间线视图
  │     ├── 按时间轴展示所有 WorkRecord
  │     ├── 支持缩放（日/周/月视图）
  │     ├── 筛选：按项目、人、类型、标签
  │     └── 点击展开详情，跳转到 Obsidian 原文
  │
  ├── 任务看板
  │     ├── 按状态分列（待办 / 进行中 / 已完成 / 已阻塞）
  │     ├── 按项目分组
  │     ├── 支持拖拽调整状态（同步回飞书 + Obsidian）
  │     └── 显示截止日、负责人、优先级
  │
  ├── 报告中心
  │     ├── 查看/生成各级别报告
  │     ├── 报告模板管理
  │     ├── 定时报告配置
  │     └── 报告导出（markdown / PDF）
  │
  ├── 数据探索
  │     ├── RAG 语义搜索
  │     ├── 结构化查询
  │     ├── 统计图表（时间分配、任务趋势、会议占比）
  │     └── 模式洞察（效率趋势、注意力分布）
  │
  └── 设置
        ├── 模型配置
        ├── 采集器管理
        ├── 存储配置
        ├── 保鲜规则配置
        ├── 报告模板和定时任务
        └── 审计数据查看和导出
```

### 设计原则

- 数据展示以 Obsidian 为底层，但提供 Obsidian 不擅长的视图
- 交互操作的结果同步回 Obsidian 和飞书
- 搜索同时覆盖结构化和语义两条路径

## 速记窗口

全局呼出的轻量窗口，不依赖主窗口，随时可用。

### 交互流程

```
1. 全局快捷键呼出（默认聚焦到输入区）
2. 输入内容（文本 + 图片混合）
3. 可选填写项目关联和标签
4. 提交 → 生成事件写入 EventLog
5. 窗口自动收起（或保持，可配置）
```

### 输入支持

- 纯文本输入
- 粘贴图片（⌘V）
- 拖拽图片/文件到窗口
- 截图（快捷键或按钮触发）
- 富文本（加粗、列表等基础格式）

### 图片处理

- 图片作为事件附件一同写入 EventLog
- 处理层对图片：小模型不能处理 → 直接升级大模型（多模态）
- 图片存储到 Obsidian 附件目录，文档中以引用方式链接

## 系统通知

```
通知类型：
  │
  ├── 需要确认（P0/P1 优先级）
  │     ├── "检测到 3 条待办，需要你确认"
  │     ├── "飞书项目状态变更需要你审批"
  │     └── "报告已生成，请查看"
  │
  ├── 轻提醒（不打断工作）
  │     ├── "今日已处理 47 条信息"
  │     └── "信息保鲜完成，2 项待确认"
  │
  └── 点击通知 → 打开主窗口对应页面
```

## 全局快捷键

```
快捷键方案（用户可自定义）：
  ⌘ + Shift + Space    呼出/收起速记窗口
  ⌘ + Shift + Enter    将当前输入作为事件提交
  ⌘ + Shift + S        截图并直接进入速记窗口
```

## 三端协同

```
场景：菜单栏看到"会议待办待确认"
  → 点击查看详情 → 打开主窗口任务看板
  → 确认/修改任务 → 同步到 Obsidian + 飞书
  → 想看原始上下文 → 跳转到 Obsidian 对应文档

场景：主窗口搜索"上个月的技术方案讨论"
  → RAG 语义搜索 → 返回相关会议记录和文档
  → 点击结果 → 在 Obsidian 中打开原文
  → 结构化筛选 → 按项目/人/时间缩小范围
```
