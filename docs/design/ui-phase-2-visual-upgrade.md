# 第二阶段：视觉升级

对齐 P0 趋势，提升视觉品质。本阶段改变应用的视觉外观。

## 前置条件

- [x] 第一阶段完成（Token 统一、颜色语义化、字体启用）

## 任务清单

### T2.1 翻转深色模式为默认

**目标**：将默认主题从浅色翻转为深色，匹配生产力工具惯例。

**方案**：
1. 修改 ThemeProvider 的 `defaultTheme` 为 `"dark"`
2. 精炼深色主题的表面层级：
   - 背景层：`oklch(13% 0 0)` — 最深
   - 卡片层：`oklch(16% 0 0)` — 中间
   - 悬浮层：`oklch(19% 0 0)` — 最浅
3. 确保 1px 边框在深色下可见（`oklch(25% 0 0)` 左右）

**设计参考**：
- Linear：`#0E0E10` 基底，通过色差分层
- Raycast：深色背景 + 毛玻璃浮窗

**验收标准**：
- [ ] 应用启动默认为深色主题
- [ ] 用户可手动切换到浅色
- [ ] 深色主题下有 3 层表面层级（背景/卡片/悬浮）
- [ ] 边框在深色下清晰可见但不突兀

**涉及文件**：
- `src/index.css` — 深色 Token 调整
- ThemeProvider 配置（`main.tsx` 或 `App.tsx`）

### T2.2 精炼深色表面层级

**目标**：建立清晰的深色模式空间层级系统。

**层级定义**：

| 层级 | 用途 | 色值建议 |
|------|------|---------|
| Layer 0 | 应用背景 | `oklch(13% 0 0)` |
| Layer 1 | 侧边栏、面板 | `oklch(15% 0 0)` |
| Layer 2 | 卡片、内容区 | `oklch(17% 0 0)` |
| Layer 3 | 悬浮、弹窗 | `oklch(20% 0 0)` |
| Layer 4 | 最顶层（模态） | `oklch(23% 0 0)` |

**方案**：
1. 定义 `--surface-0` 到 `--surface-4` Token
2. 将各组件映射到正确的层级
3. 使用 1px 边框 + 微妙阴影增强分层感

**验收标准**：
- [ ] 5 个表面层级 Token 已定义
- [ ] 各组件使用正确的层级
- [ ] 视觉上有明显的深度感

**涉及文件**：
- `src/index.css` — 新增 surface Token
- 各视图和布局组件

### T2.3 Bento Grid 仪表盘设计

**目标**：将报告页（或新建仪表盘页）重构为 Bento Grid 布局。

**设计概念**：

```
┌─────────────────────────────────────────────┐
│ ┌──────────────┐ ┌──────────────────────────┐│
│ │  今日摘要     │ │  工作时间线               ││
│ │  (小卡片)     │ │  (宽卡片)                ││
│ │  处理: 23     │ │  ████░░░░░░░░░░░░░░░░   ││
│ │  待办: 5      │ │  9:00  12:00  15:00     ││
│ └──────────────┘ └──────────────────────────┘│
│ ┌──────────────────────┐ ┌──────────────────┐│
│ │  任务概览             │ │  采集器状态       ││
│ │  (中卡片)             │ │  (小卡片)        ││
│ │  ○ 进行中: 3          │ │  ● 飞书: 活跃    ││
│ │  ◉ 已完成: 12         │ │  ○ 系统: 待激活  ││
│ └──────────────────────┘ └──────────────────┘│
│ ┌──────────────────────────────────────────┐ │
│ │  AI 摘要                                  │ │
│ │  (全宽卡片)                               │ │
│ │  今日主要完成了...                        │ │
│ └──────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**方案**：
1. 使用 CSS Grid 实现不对称网格布局
2. 每个卡片是独立的数据域（摘要、时间线、任务、采集器、AI 摘要）
3. 卡片支持 hover 微交互（轻微上浮 + 阴影加深）
4. 响应式：窄屏自动堆叠

**验收标准**：
- [ ] 仪表盘页使用 Bento Grid 布局
- [ ] 卡片有明确的数据边界和层次
- [ ] hover 状态有微交互反馈
- [ ] 窄屏下正确堆叠

**涉及文件**：
- 新建 `src/components/views/DashboardView.tsx`（或重构 ReportsView）
- 更新 `MainWindow.tsx` 导航

### T2.4 命令面板 AI 操作扩展

**目标**：将 cmdk 命令面板从简单搜索进化为 AI 驱动的操作中心。

**功能设计**：
1. **分组命令**：按类别分组（导航、操作、AI、设置）
2. **AI 建议**：基于上下文的智能操作建议
3. **快捷键提示**：每个命令右侧显示快捷键
4. **最近使用**：优先显示最近使用的命令

**命令分组**：

```
┌─ 导航 ──────────────────────────┐
│  查看事件                    ⌘1  │
│  查看任务                    ⌘2  │
│  查看时间线                  ⌘3  │
│  查看报告                    ⌘4  │
└─────────────────────────────────┘
┌─ 操作 ──────────────────────────┐
│  新建任务                        │
│  新建事件                        │
│  标记全部已读                    │
└─────────────────────────────────┘
┌─ AI ────────────────────────────┐
│  生成今日报告                    │
│  智能分类未处理事件              │
│  总结本周工作                    │
└─────────────────────────────────┘
```

**验收标准**：
- [ ] 命令按分组显示
- [ ] 每个命令显示快捷键（如有）
- [ ] AI 命令可调用后端处理
- [ ] 最近使用的命令优先显示

**涉及文件**：
- `src/components/ui/command.tsx` — 可能需要扩展
- 新建命令注册模块
- MainWindow 或独立 CommandProvider

## 风险与注意事项

1. **深色模式翻转**：可能影响用户习惯，需确保切换功能正常
2. **Bento Grid**：需要真实数据填充，避免空卡片
3. **命令面板 AI**：依赖后端 AI 能力，需确认 API 可用
4. **CaptureWindow**：独立窗口，需在 Tauri 中单独测试
