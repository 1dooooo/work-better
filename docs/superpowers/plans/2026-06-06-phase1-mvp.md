# Phase 1 MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a working Tauri desktop app with Rust backend that can collect Feishu messages via lark-cli, store events in SQLite, and display status in a menu bar UI.

**Architecture:** Tauri 2.x with Rust backend (Cargo workspace) and React 19 frontend. All business logic in Rust. Frontend communicates via Tauri commands.

**Tech Stack:** Tauri 2.x, Rust (tokio, rusqlite, reqwest, serde), React 19, TypeScript, Vite 6

**Spec:** `docs/superpowers/specs/2026-06-06-tech-selection-design.md`

```rust
use thiserror::Error;

#[derive(Error, Debug)]
pub enum WbError {
    #[error("Storage error: {0}")]
    Storage(String),
    #[error("Collector error: {0}")]
    Collector(String),
    #[error("AI error: {0}")]
    Ai(String),
    #[error("Serialization error: {0}")]
    Serialization(#[from] serde_json::Error),
    #[error("IO error: {0}")]
    Io(#[from] std::io::Error),
    #[error("Not found: {0}")]
    NotFound(String),
}

pub type Result<T> = std::result::Result<T, WbError>;
```

- [ ] **Step 1: Write Classifier**

`classify(event) -> ProcessingRoute` (Instant/Aggregate/Pattern/Archive).

Rules:
- Low confidence → Archive
- TaskUpdate, Approval, ManualNote → Instant
- Message with @mention → Instant
- Message without mention → Aggregate
- DocumentChange, Browsing, AppActivity → Aggregate
