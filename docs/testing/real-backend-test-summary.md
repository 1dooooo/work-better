## 技术决策

### 为什么选择直接调用 Rust 命令

1. **简单性**：无需启动完整 Tauri 运行时
2. **速度**：测试执行更快
3. **隔离性**：每个测试使用独立的内存数据库
4. **可维护性**：测试代码更简单，易于理解和维护

### 为什么分 crate 测试

1. **职责清晰**：每个 crate 负责特定的功能领域
2. **依赖管理**：避免不必要的跨 crate 依赖
3. **并行执行**：不同 crate 的测试可以并行运行
4. **代码组织**：测试代码与被测代码在同一 crate 中
