# PRD: 命令面板集成与 UX 改进

## Problem Statement

Work Better 当前存在以下用户体验问题：

1. **导航效率低** — 用户需要通过侧边栏手动切换视图，缺乏快速访问方式
2. **功能发现性差** — 新用户不知道有哪些功能可用，缺乏引导
3. **键盘优势未发挥** — 桌面应用的核心优势（键盘快捷键）未充分利用
4. **信息关联性弱** — 5 个独立视图缺乏关联，用户需要手动切换才能获得完整图景
5. **状态不持久** — 视图状态（过滤、排序、当前视图）刷新后丢失

## Solution

引入命令面板（⌘K）作为主要导航方式，结合工作台默认视图和状态持久化，打造键盘优先的高效工作体验。

### 核心改进

1. **命令面板** — ⌘K 触发的全局操作界面，支持导航、操作、搜索
2. **工作台视图** — 默认首页，聚合今日任务、最近事件、待处理项
3. **键盘导航** — ⌘1-5 视图切换 + ⌘K 命令面板
4. **状态持久化** — localStorage 存储用户偏好和视图状态
5. **渐进式引导** — 根据用户进度动态调整的空状态引导

## User Stories

### 命令面板核心功能

1. 作为用户，我希望按 ⌘K 打开命令面板，以便快速访问所有功能
2. 作为用户，我希望在命令面板中输入文字搜索事件，以便快速找到特定内容
3. 作为用户，我希望在命令面板中输入文字搜索任务，以便快速定位待办事项
4. 作为用户，我希望在命令面板中执行"新建任务"操作，以便快速创建任务
5. 作为用户，我希望在命令面板中执行"触发采集"操作，以便手动获取最新数据
6. 作为用户，我希望在命令面板中执行"标记事件已处理"操作，以便快速清理事件流
7. 作为用户，我希望在命令面板中切换视图（工作台、事件、任务、时间线、报告、设置），以便快速导航
8. 作为用户，我希望在命令面板中看到快捷键提示，以便学习高效操作方式

### 工作台视图

9. 作为用户，我希望打开应用时看到工作台视图，以便快速了解今日工作概览
10. 作为用户，我希望在工作台看到今日待处理任务，以便优先处理重要事项
11. 作为用户，我希望在工作台看到最近事件，以便了解最新动态
12. 作为用户，我希望在工作台看到待处理项数量，以便了解工作量
13. 作为用户，我希望在工作台看到快速操作入口，以便快速执行常用操作
14. 作为用户，我希望在工作台看到采集器状态，以便了解系统运行状况

### 键盘导航

15. 作为用户，我希望按 ⌘1 切换到工作台视图，以便快速访问首页
16. 作为用户，我希望按 ⌘2 切换到事件视图，以便查看事件流
17. 作为用户，我希望按 ⌘3 切换到任务视图，以便管理待办事项
18. 作为用户，我希望按 ⌘4 切换到时间线视图，以便查看时间线
19. 作为用户，我希望按 ⌘5 切换到报告视图，以便查看工作报告
20. 作为用户，我希望按 ⌘, 切换到设置视图，以便配置应用

### 状态持久化

21. 作为用户，我希望应用记住我上次打开的视图，以便下次直接进入
22. 作为用户，我希望应用记住事件列表的过滤条件，以便保持一致的视图
23. 作为用户，我希望应用记住任务列表的排序方式，以便保持一致的排序
24. 作为用户，我希望应用记住侧边栏的折叠状态，以便保持一致的布局

### 渐进式引导

25. 作为新用户，我希望看到清晰的空状态引导，以便了解如何开始使用
26. 作为新用户，我希望在事件空状态看到"采集"按钮，以便获取第一批事件
27. 作为新用户，我希望在任务空状态看到"新建任务"按钮，以便创建第一个任务
28. 作为用户，我希望空状态根据我的使用进度动态调整，以便获得个性化的引导

### AI 推荐（未来）

29. 作为用户，我希望在事件详情中看到"AI 建议：转为任务"，以便快速创建相关任务
30. 作为用户，我希望在任务描述中看到"AI 建议：添加标签"，以便更好地组织任务
31. 作为用户，我希望在报告生成中看到"AI 建议：添加本周总结"，以便生成更完整的报告
32. 作为用户，我希望所有 AI 推荐都需要我确认，以便保持对数据的控制权

## Implementation Decisions

### 模块变更

1. **命令面板模块** (`src/components/ui/command.tsx`)
   - 更新 `CommandDialog` 组件，集成 Variant B 设计
   - 添加图标容器样式
   - 实现分组：快速导航、常用操作、AI 推荐、搜索结果
   - 连接真实数据源

2. **主窗口模块** (`src/components/layout/MainWindow.tsx`)
   - 添加命令面板触发逻辑
   - 集成快捷键处理
   - 更新默认视图为工作台

3. **键盘快捷键模块** (`src/hooks/useKeyboardShortcuts.ts`)
   - 确保 ⌘K、⌘1-5、⌘,、⌘N 正常工作
   - 添加快捷键提示格式化

4. **状态持久化模块** (新增 `src/hooks/useStatePersistence.ts`)
   - 实现 localStorage 读写
   - 支持视图状态、过滤条件、排序方式持久化

5. **数据获取模块** (新增 `src/hooks/useCommandData.ts`)
   - 提供事件、任务、设置数据
   - 支持模糊搜索

### 接口设计

**命令面板接口**：
```typescript
interface CommandItem {
  id: string;
  label: string;
  icon: LucideIcon;
  shortcut?: string;
  action: () => void;
  category: 'navigation' | 'action' | 'ai' | 'search';
}

interface CommandGroup {
  label: string;
  items: CommandItem[];
}
```

**状态持久化接口**：
```typescript
interface PersistedState {
  lastView: ViewId;
  eventFilter: FilterSource;
  taskSort: SortField;
  sidebarCollapsed: boolean;
}
```

### 架构决策

1. **命令面板采用 Variant B 设计** — 分类图标布局，视觉层级清晰
2. **状态持久化使用 localStorage** — Tauri 桌面应用不需要 URL 分享功能
3. **默认视图改为工作台** — 降低新用户学习成本
4. **AI 推荐采用主动推荐策略** — AI 根据上下文推荐，用户确认后执行

## Testing Decisions

### 测试原则

- **只测试外部行为**，不测试实现细节
- **测试用户可见的功能**，不测试内部状态
- **测试边界情况**，如空状态、错误状态、加载状态

### 测试模块

1. **命令面板测试** (`src/components/ui/command.test.tsx`)
   - 测试打开/关闭功能
   - 测试搜索功能
   - 测试命令执行
   - 测试快捷键触发

2. **键盘快捷键测试** (`src/hooks/useKeyboardShortcuts.test.ts`)
   - 测试快捷键匹配
   - 测试快捷键触发
   - 测试输入框内快捷键行为

3. **状态持久化测试** (`src/hooks/useStatePersistence.test.ts`)
   - 测试状态保存
   - 测试状态恢复
   - 测试默认值处理

4. **数据获取测试** (`src/hooks/useCommandData.test.ts`)
   - 测试数据加载
   - 测试搜索功能
   - 测试错误处理

### 测试覆盖率目标

- **单元测试覆盖率**：≥ 80%
- **集成测试覆盖率**：≥ 70%
- **E2E 测试覆盖**：核心用户流程

## Out of Scope

### 本期不实现

1. **AI 推荐功能** — 作为未来迭代方向，本期只实现基础框架
2. **插件化架构** — 作为未来扩展方向，本期不实现
3. **协作功能** — 作为未来功能，本期不实现
4. **高级动画** — 弹簧物理动画作为体验提升，本期只实现基础过渡
5. **全键盘操作** — 只实现基础导航（⌘1-5 + ⌘K），不实现列表内 vim 风格导航

### 技术限制

1. **不修改现有采集器逻辑** — 只集成到命令面板，不改变采集器实现
2. **不修改现有任务管理逻辑** — 只集成到命令面板，不改变任务管理实现
3. **不修改现有报告生成逻辑** — 只集成到命令面板，不改变报告生成实现

### 风险和注意事项

#### 潜在风险
1. **数据源集成** — 可能需要调整数据获取逻辑
2. **性能问题** — 大量数据可能导致命令面板卡顿
3. **主题兼容性** — 图标容器样式在深色/浅色主题下可能需要调整

#### 注意事项
1. **保持现有功能** — 不要破坏现有的命令面板功能
2. **渐进式集成** — 可以先集成导航和操作，再添加 AI 推荐
3. **测试覆盖** — 确保新功能有充分的测试覆盖
